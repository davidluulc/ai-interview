# 部署与中间件

## 当前阶段

当前项目还没有做真实云服务器上线，但已经为上线做了准备：

- FastAPI 后端；
- SQLite 本地数据库；
- 用户系统；
- RAG / Agent 日志；
- 管理员后台；
- 测试体系；
- 部署准备文档。

## 上线时整体链路

真实上线大致是：

```text
浏览器
-> 域名
-> Nginx
-> FastAPI / Uvicorn
-> 数据库
-> Redis
-> 模型厂商 API
-> 文件/对象存储
```

## Uvicorn

Uvicorn 是 Python ASGI 服务器。

它不是云服务器，而是运行 FastAPI 应用的进程。

云服务器是机器，Uvicorn 是机器上跑后端代码的服务程序。

## Nginx

Nginx 通常站在公网入口。

它负责：

- 接收外部请求；
- 反向代理到 FastAPI；
- 配置 HTTPS；
- 静态文件；
- 限流；
- 超时控制。

反向代理可以理解为：

```text
用户不知道后端真实端口
只访问 Nginx
Nginx 再把请求转发给内部后端服务
```

## Docker

Docker 用来把应用和运行环境打包。

好处：

- 本地和服务器环境一致；
- 部署更稳定；
- 方便启动多个服务；
- 方便 docker-compose 管理后端、数据库、Redis。

## 数据库

本地可以用 SQLite。

上线更常见：

- MySQL；
- PostgreSQL。

如果后续做向量检索，可以考虑：

- PostgreSQL + pgvector；
- Qdrant；
- Milvus。

## Redis

Redis 可用于：

- refresh token 黑名单；
- 缓存；
- 限流；
- 任务队列；
- 临时会话状态。

当前项目还没有强依赖 Redis，后续上线和高并发时再引入更合适。

## 面试表达

```text
当前项目主要完成了应用功能和工程化基础，还没有真实云部署。
如果上线，我会用 Nginx 作为公网入口，反向代理到 Uvicorn 跑的 FastAPI 服务。
数据库从 SQLite 迁移到 MySQL 或 PostgreSQL，Redis 用于缓存、限流或 token 管理。
后续如果向量数据量变大，再把 SQLite embedding 迁移到 Qdrant 或 pgvector。
```
