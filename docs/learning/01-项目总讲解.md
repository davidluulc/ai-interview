# 项目总讲解

## 项目一句话

这是一个面向大学生和求职者的 AI 模拟面试系统。用户可以上传简历、输入目标岗位 JD，系统结合岗位知识库 RAG、题库 RAG、候选人画像 RAG 生成面试问题，并由面试 Agent 根据用户回答动态调整追问策略，最后生成报告、复盘和训练建议。

## 核心流程

```text
用户资料输入
-> 简历 / JD / 岗位信息进入候选人档案
-> 三个 RAG 检索上下文
-> Agent 构造当前状态
-> Agent Decision 决定下一步动作
-> LLM 生成下一道面试问题
-> 用户回答
-> 更新历史、候选人画像、日志和训练建议
```

## 三个 RAG

- 岗位知识库 RAG
  - 存岗位 JD、技术栈、业务知识、面试考察点。
- 题库 RAG
  - 存不同岗位、不同难度、不同阶段的面试题和参考要点。
- 候选人画像 RAG
  - 存简历概况、历史回答、薄弱点、训练记录。

## Agent 做什么

Agent 不是单纯问答。它会观察：

- 当前轮次；
- 上一轮问答；
- 历史所有问答；
- 三个 RAG 的命中情况；
- 候选人薄弱点；
- coach / interview 模式。

然后决定：

- 继续深挖；
- 降低难度；
- 切换话题；
- 给学习引导；
- 结束面试。

## 项目亮点

- 用户系统和登录鉴权；
- 简历 / 岗位档案；
- 三类 RAG 协作；
- Agent 决策日志；
- RAG 命中日志；
- 低质量召回后台面板；
- 训练任务闭环；
- 后续可迁移 LangGraph、Qdrant、Docker/Nginx 上线。

## 面试表达

你可以这样说：

```text
我做的是一个 AI 模拟面试系统，不是简单调用大模型问答。
系统会根据用户简历、岗位 JD 和目标岗位，分别检索岗位知识库、题库和候选人画像三个 RAG。
检索结果会进入 Agent State，Agent 根据当前轮次、历史回答、薄弱点和召回质量做决策，再让大模型生成下一道问题。
同时系统会记录 RAG 命中日志和 Agent 决策日志，方便排查为什么问这个问题、用了哪些资料、召回质量如何。
```
