# Roadmap 说明

## 当前入口

开发路线判断优先看：

```text
docs/roadmap/current-state.md
```

这个文件以当前代码、测试和数据库模型为准，负责回答：

- 已经真实落地了什么。
- 哪些只是文档规划，尚未落地。
- 哪些旧路线不要重复执行。
- 下一阶段最合理的开发方向是什么。

## 历史进度

`project-progress.md` 是历史执行记录，用于查证过去每轮做了什么、跑过哪些测试、有哪些风险。

它不是最新路线总纲。如果它里面早期段落的“下一步建议”和 `current-state.md` 冲突，以 `current-state.md` 为准。

## 部署资料

- `deployment-tech-selection.md`
- `deployment-preflight-checklist.md`

这两个文件用于部署阶段参考，不代表当前已经进入真实上线。
