# 项目演示脚本

## 1. 登录和档案

登录系统，创建投递档案，录入简历、岗位 JD 和公司信息。

讲解重点：

- 档案是后续 RAG 和 Agent 的用户上下文。
- 同一个用户可以针对不同岗位建立不同投递档案。

## 2. 开始面试

进入面试页面，系统结合三类 RAG 和候选人档案生成第一道问题。

讲解重点：

- 岗位知识库让问题贴合岗位。
- 题库 RAG 让问题更接近真实面试。
- 候选人画像 RAG 让问题结合简历和历史表现。

## 3. 动态追问

用户回答后，后端构造 agent state，LangGraph/Agent 根据历史回答、RAG 命中和回答质量决定下一步动作。

讲解重点：

- Agent 不只是生成文本，而是先决策再生成。
- 决策可能是追问、降难度、切换话题或结束面试。
- runtime audit 和 workflow trace 可以解释链路。

## 4. 查看报告

面试结束后查看复盘报告、weakTags 和逐题反馈。

讲解重点：

- 报告不是简单总结，而是连接后续训练任务。
- weakTags 用于定位用户短板。

## 5. 训练闭环

进入训练中心，根据 weakTag 打开专项练习，提交练习后更新掌握度。

讲解重点：

- 项目从“模拟面试”延伸到“训练提升”。
- 用户可以针对薄弱点反复练习。

## 6. 管理员后台

展示 RAG 命中情况、Agent workflow 观测、基础设施状态和 RAG ingestion 任务状态。

讲解重点：

- RAG 质量诊断用于排查召回问题。
- Agent 工作流观测用于排查决策问题。
- 异步任务 Worker 状态用于解释慢任务处理能力。
- 基础设施状态说明项目具备生产化部署路径。

## 7. 结束表达

这个项目的核心不是简单调用大模型，而是把 RAG、Agent 工作流、训练闭环和工程化可观测性串成一个完整 AI 应用。它既能服务用户模拟面试，也能让开发者定位系统为什么这样提问、哪里召回不好、任务为什么失败。
