# 阶段一执行计划：面试体验增强 V2

## 1. 目标

本计划用于执行总路线 Spec 的阶段一：

```text
面试体验增强。
```

目标是让 AI 模拟面试系统从“能生成面试问题”升级为“能帮助用户训练面试”。

阶段一不追求一次性升级所有 Agent 和 RAG 能力，而是优先改善用户能直接感受到的体验：

- 问题不要机械重复。
- 学习辅导模式和真实面试模式要有明显差异。
- 每道题要有考察点和追问依据。
- 用户连续答不上来时，系统要能降难度或切换话题。
- 报告要能指导下一轮训练。

## 2. 当前基础

当前项目已经具备：

- `agentMode`：学习辅导模式 / 真实面试模式。
- `AgentDecision`：包含下一步动作、难度、原因、触发规则。
- 前端 Agent 模式选择。
- 前端 Agent 决策依据展示。
- 逐题复盘报告。
- Agent 决策日志。
- RAG 命中日志。

阶段一是在这些能力上继续增强，而不是从零开始。

## 3. 核心改动方向

### 3.1 面试问题生成增强

目标：

- 问题更贴合简历、JD、当前回答和 Agent 决策。
- 问题不能只围绕一个知识点机械深挖。
- 问题标题和问题内容必须一致。

建议改动：

- 后端 prompt 中明确要求：
  - `focus` 必须是具体考察点。
  - `prompt` 必须和 `focus` 对齐。
  - 一次只问一个问题。
  - 如果上一轮回答明显不会，不继续要求写复杂示例。
- 增加问题重复检测：
  - 后端检测已问问题。
  - 前端保留本地兜底检测。
  - 连续相似问题触发 `switch_topic`。

### 3.2 学习辅导模式增强

目标：

学习辅导模式不是“真实面试官继续拷问”，而是像教练一样帮助用户补基础。

建议行为：

- 用户第一次答不上来：
  - 降低难度。
  - 追问基础概念。
- 用户连续两次答不上来：
  - 给出回答框架提示。
  - 问一个更小的问题。
- 用户连续三次答不上来：
  - 切换话题。
  - 在报告中标记该知识点为训练任务。

示例：

```text
不要继续问：
请写出完整 RAG 日志 JSON。

应该改成：
我们先拆小一点：RAG 命中日志通常要记录“用户问了什么”和“检索到了什么”，你能先说出这两个字段分别对应什么吗？
```

### 3.3 真实面试模式增强

目标：

真实面试模式要有压力，但不能无意义卡死。

建议行为：

- 用户答得完整：
  - 深挖实现细节。
  - 提高难度。
- 用户答得模糊：
  - 要求补项目例子。
  - 要求解释取舍。
- 用户答不上来：
  - 先追问基础。
  - 连续答不上来后切换到相邻话题。

真实面试模式可以保留压力表达，但不能羞辱用户。

### 3.4 报告增强

目标：

报告要从“分数 + 建议”升级为“下一轮训练计划”。

建议新增：

- 本轮薄弱知识点。
- 下一轮优先训练顺序。
- 每个薄弱点的 1 分钟回答模板。
- 推荐练习题。
- 是否建议重做本轮面试。

报告中应区分：

- 当前答案问题。
- 需要补的技术知识。
- 需要补的项目表达。
- 下一轮训练动作。

### 3.5 前端体验增强

目标：

让用户看得懂当前系统为什么这么问。

建议改动：

- 当前问题区展示：
  - Agent 模式。
  - 考察点。
  - 追问依据。
  - 下一步动作。
- 报告区展示：
  - 逐题复盘。
  - 训练任务。
  - 建议重练方向。
- Debug 区分层：
  - 普通用户看解释。
  - 开发者看 RAG 和 Agent 日志。

## 6. 验收标准

阶段一完成后必须满足：

- 学习辅导模式和真实面试模式在问题风格上有明显差异。
- 用户连续答不上来时不会一直被要求回答同一个复杂问题。
- 每道题能展示考察点或追问依据。
- Agent 决策日志能看到触发规则。
- 面试报告能输出下一轮训练计划。
- 后端全量测试通过。
- 前端测试脚本通过。
- 用户完成一次面试后，能根据报告知道下一步该练什么。

## 7. 学习重点

阶段一开发时，你需要重点理解：

- Prompt 约束为什么影响模型稳定性。
- Agent State 和 Agent Decision 的区别。
- 为什么要记录 Agent 决策日志。
- 为什么连续弱回答要切换策略。
- 前端为什么要展示“追问依据”。
- 报告为什么要从评价变成训练计划。

## 8. 面试表达

可以这样讲阶段一：

> 我在项目里把面试体验分成学习辅导模式和真实面试模式。学习辅导模式更像教练，用户连续答不上来时会降低难度、拆小问题或切换到基础解释；真实面试模式保留一定压力，但也会避免无效重复追问。后端 Agent 会根据回答状态、连续弱回答次数、剩余轮次和 RAG 命中质量生成结构化决策，并把触发规则写入日志。前端会展示当前问题的考察点和追问依据，报告会给出逐题复盘和下一轮训练动作。
